1. Executive Summary

LaaS (Lab as a Service) is a unified, self-service web portal that consolidates the management of eight enterprise infrastructure platforms into a single browser-based interface. Built by the Wipro SDX Infrastructure & Data Centre team, it eliminates the need for engineers and managers to access individual platform consoles, enforcing role-based governance while dramatically accelerating day-to-day operations.

2. New in v6.2 — Insights & Analytics Engine

The flagship addition in v6.2 is the Insights & Analytics page, backed by a PostgreSQL 16 daily snapshot database. An automated collector runs every night at 11:00 PM, capturing metrics from all eight platforms into twelve dedicated tables with a thirty-day rolling retention window.

🩺 Health Scorecard
  • Real-time RAG (Red / Amber / Green) status across all eight platforms
  • Trend arrows showing improvement or degradation vs previous day
  • Per-platform health detail: CPU%, Memory%, storage, alerts
  • Single-glance dashboard for NOC and management reviews
🔀 Change Detection
  • Automatic day-over-day drift alerts
  • Tracks VM count, host count, pod count, IP utilisation shifts
  • Highlights newly added or removed resources
  • Colour-coded severity: informational, warning, critical
💰 Capacity & Cost
  • Storage and compute exhaustion date predictions
  • Monthly chargeback figures in INR (₹) and USD ($)
  • Per-platform resource consumption breakdown
  • Cost trend over the rolling thirty-day window
📋 Executive Summary
  • One-page, PDF-ready overview with aggregate KPIs
  • Platform health matrix suitable for management sharing
  • Key risk areas and capacity warnings highlighted
  • Suitable for weekly status reports and steering committee reviews
📈 Historical Trending & Capacity Forecasting
  • Interactive trend charts for 7, 14, and 30-day windows covering all collected metrics
  • VM count, host count, pod count, and IP utilisation trends visualised over time
  • Linear regression capacity forecasting per metric with exhaustion date predictions
  • Storage, compute, and network exhaustion early-warning indicators
  • Day-over-day change arrows with percentage shift values
  • CSV-exportable historical data snapshots for offline analysis
🗄️ PostgreSQL Daily Snapshot Infrastructure
  • PostgreSQL 16 database with twelve dedicated snapshot tables
  • Automated Task Scheduler job runs daily at 11:00 PM
  • Captures: VMware, Hyper-V, Nutanix, OpenShift, AWS, IPAM, AD/DNS, Project Utilisation
  • Thirty-day rolling retention with automatic purge of older records
  • Retry logic with exponential back-off for resilient multi-platform collection
  • Three database views and a purge function for data maintenance

5. Access Control & Roles

Role Description Key Permissions
Admin Full control of all platforms and portal settings All platform tabs · Add/edit/delete infrastructure sources · Approve/decline VM requests · Manage portal users & roles · Configure pricing, thresholds, notifications · View & export audit log
Operator Day-to-day operations and provisioning approvals All VMware VM actions · OCP node cordon/drain · Nutanix VM actions & snapshots · Approve/decline VM requests · AD user/group/DNS management · Launch Ansible templates · View IPAM, Assets, Chargeback
Viewer Read-only visibility across all platforms All monitoring tabs (read-only) · IPAM, Assets, Chargeback (read-only) · Ansible: view templates/jobs (no launch) · Full audit log access (view only)
Requester Self-service VM provisioning requests only Submit VMware / OpenShift / Nutanix VM requests · Track own request status · View personal audit history · No infrastructure monitoring access
