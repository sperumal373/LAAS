01Executive Summary

Overview
LaaS (Lab as a Service) Dashboard is a self-hosted internal web portal built for managing multi-platform lab infrastructure across the SDX Data Centre and Disaster Recovery sites. It provides a unified interface for VMware vSphere, Red Hat OpenShift, Nutanix Prism Central, Ansible Automation Platform, Amazon Web Services, Microsoft Hyper-V, Enterprise Storage (Pure, Dell, HPE, NetApp), Backup & Data Protection (Rubrik, Cohesity, Veeam), IPAM, CMDB, Active Directory & DNS, and Insights & Analytics — all from a single browser window.

The portal follows a three-tier architecture: a React 18 single-page application (SPA) served over HTTPS handles all user interactions, a FastAPI Python backend (~6,400 lines) exposes a secure REST API integrating with 8+ platform APIs, and a dual-database layer using SQLite (operational data) and PostgreSQL 16 (analytics & CMDB). All tiers run as NSSM-managed Windows services on the same host, accessible at https://caas-dashboard.sdxtest.local.

07Authentication Flow

StepMethodDescription
1 Hardcoded Admin A bootstrap administrator account is checked first. This guarantees portal access before any users are provisioned in the database. Credentials are defined in the backend configuration.
2 Local DB User Passwords are stored as PBKDF2-SHA256 hashes (with salt) in SQLite. Local users are created by an Admin through the User Management page. Ideal for service accounts or non-AD users.
3 Active Directory LDAP bind performed against the corporate Active Directory. On first successful login, the user record is automatically created or updated (upserted) in SQLite. The email address is captured from the AD profile and stored locally.

08Role-Based Access Control

RoleAccess LevelCapabilities
Admin Full Access All navigation pages, user management (create, edit, delete, assign roles), approve or decline VM provisioning requests, view all users audit logs, manage VMware tag owners, configure pricing and vCenter connections.
Operator Operational All infrastructure pages (VMs, Capacity, Snapshots, Networks, Project Utilization), approve or decline VM requests, view audit logs. Cannot manage user accounts or change role assignments.
Viewer Read-Only Overview dashboard, Virtual Machines, Project Utilization, Capacity, Snapshots, and Networks pages in read-only mode. Cannot submit VM requests, access audit logs, or modify any data.
Requester Self-Service Submit new VM provisioning requests, track own request status, and view personal audit trail. No access to infrastructure dashboards or other users data.

13Daily Collector & Insights Architecture

The Daily Collector (daily_collector.py) runs as a scheduled task at 23:00 daily. It connects to all eight platforms, collects metrics, and inserts daily snapshot rows into PostgreSQL 16 (port 5433, database caas_dashboard).

AspectDetail
Tables Populateddaily_snapshots, daily_vm_snapshots, daily_host_snapshots, daily_ds_snapshots, daily_ocp_snapshots, daily_nutanix_snapshots, daily_project_snapshots, chargeback_snapshots, ipam_vlans, ipam_ips
Retention30-day rolling window with automatic purge of older records
ResilienceRetry logic with exponential back-off for transient failures across all platform APIs
ScheduleWindows Task Scheduler: 23:00 daily, runs daily_collector.py via the backend Python venv
