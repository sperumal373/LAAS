1. Introduction & About LaaS Portal
What is LaaS and why it exists
The LaaS Portal (Lab as a Service) is Wipro's unified infrastructure management platform for SDX INFRA & DC Operations. It provides a single-pane-of-glass web dashboard to manage, monitor, and operate multi-cloud and on-premises resources across VMware vCenter, Red Hat OpenShift, Nutanix AHV, and Ansible Automation Platform.

The portal eliminates the need to log in to individual vCenter/OCP/Nutanix consoles for routine operations. It consolidates health monitoring, VM lifecycle management, capacity reporting, IP address management, chargeback, and audit logging into a single role-based access-controlled interface accessible from any modern browser.
🖥️
Unified Multi-Platform View
Single dashboard for VMware, OpenShift, Nutanix, and Ansible — no separate logins needed.
⚡
VM Lifecycle Management
Power on/off, clone, migrate, snapshot, and delete VMs directly from the portal.
📊
Capacity Planning
CPU, RAM, and storage utilization trends with forecasting gauges across all environments.
🔒
Role-Based Access Control
Admin, Operator, Viewer, and Requester roles with granular permission enforcement.
📋
VM Request & Approval Workflow
Requesters raise VM provisioning requests; Admins approve or reject with full audit trail.
💳
Automated Chargeback
Project-based cost allocation with monthly PDF/CSV export and visualized cost breakdown.
📡
IPAM Integration
Live IP address management — subnet occupancy, free IPs, and allocation history.
🔍
Full Audit Log
Every action is logged with user, timestamp, and affected resource for compliance reporting.

3. Quick Start Guide
Get up and running in 5 minutes
Getting Started
1
Open the Portal
Navigate to http://<server-ip> in your browser. Ensure you are connected to the corporate network or VPN.
2
Log In
Enter your username and password on the login screen. If you don't have credentials, contact your Admin. Default demo credentials: admin / admin123.
3
Select Environment
Use the vCenter dropdown in the top bar to choose a specific vCenter or "All vCenters" for an aggregated view. The Overview dashboard updates automatically.
4
Explore the Dashboard
The Overview page shows your SDx Infrastructure banner with live counts for VMs, hosts, OCP nodes, pods, and Nutanix resources. KPI cards show current utilisation.
5
Navigate by Platform
Use the left sidebar to switch between VMware, OpenShift, Nutanix, Ansible, IPAM, and other modules. Snapshots appears nested under VMware.
💡
Tip: Press Ctrl+K or use the 🔍 global search bar to instantly jump to any VM, snapshot, OCP cluster, or menu item.

7. Snapshot Management
Nested under VMware — manage VM snapshots
The Snapshots page (nested under VMware in the sidebar) provides a centralized view of all VM snapshots across all managed vCenters.

Columns: VM Name · Snapshot Name · Creation Date · Age (days) · Size (if available) · vCenter

Actions available (Admin & Operator):
  • Delete individual snapshot from the snapshot row
  • Bulk-select and delete multiple snapshots at once
  • Create a new snapshot from any VM on the VMware page
ℹ️
Best Practice: Aged snapshots (>7 days) consume significant datastore space and degrade VM performance. Review and clean up snapshots regularly.

11. Capacity Planning
Resource utilisation monitoring and forecasting
The Capacity page provides detailed CPU, RAM, and Storage utilisation data for each vCenter:
  • Utilisation Gauges — animated arc gauges showing used%, with color thresholds (green <60%, yellow 60–80%, red >80%)
  • ESXi Host Table — per-host CPU/RAM free, VM count, and power state
  • Datastore Table — datastore name, type, capacity, free space, and usage bar
  • Trend Charts — 7/14/30-day historical trend lines for CPU and RAM utilisation
⚠️
Action Required threshold: When any resource exceeds 80% utilisation, the gauge turns red and a warning banner appears. Notify your infrastructure team when this occurs.

15. VM Request Workflow
Self-service VM provisioning with approval
Workflow Steps
1
Requester Raises a Request
Navigate to VM Requests → New Request. Fill in VM Name, vCPU, RAM, Storage, OS, Project/Cost Center, justification. Submit.
2
Admin Receives Notification
Admin sees the pending badge counter on the VM Requests nav item. Email notification is also triggered (if configured).
3
Admin Reviews & Decides
Admin opens the request, reviews justification and resource specs, then clicks Approve or Reject with optional comments.
4
VM Provisioned or Rejected
If approved, the VM is provisioned from a predefined template. If rejected, the requester sees the reason in their request history.

16. Audit Log
Compliance and traceability — every action logged
Every user action performed through the portal is recorded in the Audit Log with:

• Timestamp — Date and time (UTC) of the action
• User — Who performed the action
• Action Type — e.g. VM_POWER_ON · VM_DELETE · SNAPSHOT_CREATE · USER_LOGIN
• Resource — Which VM, host, or object was affected
• Status — Success or Failure
• Details — Additional context / parameters

Filters: Date range · User · Action type · Status · Free text search
Export: CSV export available (Admin only)

Architecture, stack, server specs, and API reference

Frontend → React 18 SPA (Vite build) served as static files
Backend API → FastAPI (Python 3.11+) with SQLite database
Data Sources → VMware vCenter REST API · OCP API + Prometheus · Nutanix Prism Central API · Ansible AAP API · phpIPAM REST API

🔒 Security Notes
• All API endpoints are protected by JWT-based session authentication
• Role checks are enforced server-side — UI hiding is supplementary only
• SQLite database is stored locally; backup regularly to a network share
• vCenter/OCP/Nutanix credentials are stored encrypted in the DB
• HTTPS (TLS) is recommended for production deployment behind a reverse proxy (nginx)

19. FAQ & Troubleshooting
Common questions and how to resolve them
Q: The portal shows "0 VMs" but vCenter has VMs — why?
Check that the vCenter is correctly registered under Settings and the credentials are valid. Try clicking Retry/Refresh on the Overview page. If the issue persists, check the backend logs for vCenter API errors.
Q: I cannot see the Audit Log or Users menu items
Audit Log requires Operator or Admin role. Users & Roles requires Admin role. Contact your Admin to upgrade your role if needed.
Q: Snapshot creation timed out — what should I do?
Large VMs or datastores with high I/O can cause snapshot operations to time longer than the portal's timeout. Check the vCenter task list directly to see if the snapshot completed. If it failed, retry during off-peak hours.
Q: The OCP cluster shows "Degraded" even though the cluster seems healthy
The portal flags Degraded if more than 25% of nodes are NotReady OR more than 15% of Cluster Operators are degraded. Check the Nodes and Operators tabs in the OpenShift module for details. Transient operator cycling (e.g. during upgrades) can trigger this briefly.
Q: How do I add a new vCenter / OCP cluster / Nutanix Prism Central?
Navigate to Settings (Admin only) → Add vCenter / Add OCP Cluster / Add Nutanix PC. Enter the API endpoint URL, credentials (stored encrypted), and save. The portal will begin polling the new environment immediately.
Q: Can I export VM lists or capacity reports?
Yes — most tables have a CSV export button (available to Admin and Operator). Chargeback also has a PDF export. For audit log export, Admin role is required.
Q: The analog clock in the sidebar shows the wrong time
The clock is driven by the browser's local system clock, not the server. Ensure your workstation clock is synchronized to the corporate NTP server.
